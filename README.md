# КАРТОНКА v3

Telegram Mini App для взаємодопомоги на мирних акціях.

## Що працює

- створення точки підтримки без переходу в Telegram-чат;
- обов’язкове фото точки;
- точне місце на карті OpenStreetMap;
- перелік ресурсів і поточна кількість;
- час, до якого власник контролює точку;
- редагування запасів, адреси, фото та часу роботи;
- автоматичне зникнення прострочених точок;
- кнопка «Точки більше немає»;
- автоматичне приховування після 20 унікальних повідомлень;
- підтвердження «Точка працює»;
- створення та редагування потреб усередині застосунку;
- контакт автора потреби показується тільки після натискання «Я допоможу»;
- резервні коди для відновлення керування;
- зберігання кодів керування в Telegram SecureStorage, коли воно доступне.

## 1. Створи Supabase

1. Відкрий Supabase та створи новий проєкт.
2. У Dashboard відкрий **SQL Editor → New query**.
3. Скопіюй увесь файл `supabase/setup.sql` і натисни **Run**.
4. Переконайся, що в **Storage** з’явився публічний bucket `point-photos`.

## 2. Заповни config.js

У Supabase відкрий **Settings → API** та скопіюй:

- Project URL;
- anon / publishable key.

Встав їх у `config.js`:

```js
supabaseUrl: 'https://YOUR_PROJECT.supabase.co',
supabaseAnonKey: 'YOUR_ANON_KEY',
```

Anon key можна зберігати у фронтенді. **Service role key ніколи не додавай у GitHub.** Захист забезпечують RLS і RPC-функції з `setup.sql`.

## 3. Онови GitHub

Замінити потрібно весь вміст репозиторію файлами з архіву v3, включно з:

- `app.js`;
- `db.js`;

Після commit дочекайся зеленого статусу GitHub Pages, повністю закрий Mini App і відкрий його знову.

## Як працює керування точкою

Під час створення застосунок генерує випадковий секретний код. Він:

- зберігається у Telegram SecureStorage або локально на пристрої;
- не потрапляє в публічний список;
- перевіряється базою у вигляді SHA-256 хешу;
- дозволяє редагувати або закривати тільки власну точку.

Після створення показується резервний код. Його можна вставити у **Мої дії → Відновити доступ за кодом**.

## Як працюють повідомлення «точки немає»

Один Telegram-користувач або один браузерний профіль може залишити одне повідомлення для конкретної версії точки. Після 20 повідомлень точка отримує статус `hidden` і більше не повертається у публічному списку. Коли власник редагує точку, лічильник скидається, а точка знову стає активною.

## Важливо для публічного запуску

Поточна версія є робочим MVP. Перед великим публічним запуском бажано додати серверну перевірку Telegram `initData`, rate limiting і модерацію фотографій. Не публікуй токен Telegram-бота у GitHub або в коді Mini App.
