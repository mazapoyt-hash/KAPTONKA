# КАРТОНКА

**КАРТОНКА** — Telegram Mini App і PWA для взаємодопомоги під час мирних акцій.

> У кожного протесту має бути тил.

## Що вже працює

- список актуальних потреб із категоріями та прогресом;
- точки підтримки: вода, зарядка, туалет, картон, медична допомога тощо;
- фільтри за містом і категорією;
- сортування точок за відстанню після дозволу користувача;
- кнопка «Я допоможу» і локальний список особистих дій;
- повідомлення про нову потребу або точку підтримки;
- Telegram Share та адаптація до теми Telegram;
- PWA та кешування для нестабільного інтернету;
- локальний редактор JSON через `admin.html`;
- автоматичний деплой на GitHub Pages.

## Безпека токена

**Ніколи не додавай токен Telegram-бота у файли репозиторію.** Якщо токен уже надсилався у чат або був опублікований, відклич його через:

`@BotFather → /mybots → бот → API Token → Revoke current token`

Для цієї статичної Mini App токен взагалі не потрібен. Він знадобиться лише пізніше для серверного бота або webhook і має зберігатися в секретах хостингу.

## 1. Завантаження на GitHub

1. Створи новий репозиторій.
2. Розпакуй ZIP та завантаж **усі файли з папки проєкту в корінь репозиторію**.
3. Відкрий `Settings → Pages`.
4. У `Build and deployment → Source` обери **GitHub Actions**.
5. Після виконання workflow отримаєш адресу приблизно такого вигляду:

```text
https://USERNAME.github.io/REPOSITORY/
```

## 2. Налаштування застосунку

Відкрий `config.js` і зміни:

```js
botUsername: 'USERNAME_БОТА',
coordinatorTelegram: 'https://t.me/КАНАЛ_АБО_КООРДИНАТОР',
publicAppUrl: 'https://USERNAME.github.io/REPOSITORY/',
demoMode: false,
```

`demoMode: false` став лише після того, як заміниш демонстраційні дані на реальні перевірені.

## 3. Дані

Усі публічні дані знаходяться в папці `data`:

- `data/needs.json` — потреби;
- `data/points.json` — точки підтримки;
- `data/updates.json` — оголошення.

Після зміни JSON зроби commit — GitHub Pages оновиться автоматично.

### Вбудований редактор

Після запуску відкрий:

```text
https://АДРЕСА_ЗАСТОСУНКУ/admin.html
```

Відредагуй дані, натисни «Завантажити JSON» і заміни відповідний файл на GitHub. Це локальний редактор, він не є захищеною адмін-панеллю та не змінює сайт самостійно.

## 4. Підключення до Telegram

1. Відкрий `@BotFather`.
2. Створи бота командою `/newbot`, якщо його ще немає.
3. Відкрий `/mybots` і вибери бота.
4. Перейди до `Bot Settings → Configure Mini App` або налаштуй кнопку меню.
5. Вкажи HTTPS-адресу GitHub Pages.
6. Назва кнопки, наприклад: **Відкрити КАРТОНКУ**.

## 5. Логотип

- повний логотип: `assets/logo.png`;
- іконки застосунку: `assets/icon-192.png`, `assets/icon-512.png`;
- favicon: `assets/favicon.svg`.

## Важливі принципи

- не публікуй точне переміщення учасників;
- не збирай списки людей, які прийшли на акцію;
- усі потреби та точки мають проходити перевірку координаторами;
- за можливості спрямовуй оплату напряму постачальнику, магазину або службі доставки;
- не публікуй приватні телефони та адреси без згоди власника;
- видаляй або позначай застарілі потреби одразу після їх закриття.

## Наступний етап

Для оновлень у реальному часі можна підключити Supabase або інший backend. Поточна версія спеціально працює без сервера, щоб її можна було швидко запустити через GitHub Pages.
