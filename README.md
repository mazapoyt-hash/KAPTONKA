# КАРТОНКА

**КАРТОНКА** — Telegram Mini App і PWA для взаємодопомоги під час мирних акцій.

> У кожної мирної акції має бути тил.

## Що виправлено у версії 2.0

- логотип у шапці вбудований безпосередньо в HTML і більше не залежить від зовнішнього файлу;
- іконки перенесено в корінь проєкту, щоб GitHub Pages коректно їх публікував;
- демонстраційний режим вимкнено;
- усі вигадані потреби й точки видалено з публічних даних;
- порожні розділи показують зрозумілий стан, а не тестові картки;
- виправлено зайвий HTML-тег у фільтрі потреб;
- форми створення потреби й точки копіюють текст і відкривають системне меню поширення або Telegram;
- посилання перевіряються перед відкриттям;
- оновлено офлайн-кеш і механізм його очищення;
- додано стабільне кешування JSON для роботи без мережі;
- посилання застосунку вже налаштоване на `https://mazapoyt-hash.github.io/KAPTONKA/`.

## Як оновити сайт

1. Розпакуй ZIP.
2. Видали старі файли в репозиторії або завантаж нові з підтвердженням заміни.
3. Важливо завантажити **весь вміст папки в корінь репозиторію**, включно з `.github`, `data`, `examples` та файлами іконок.
4. Дочекайся зеленого статусу workflow `Deploy GitHub Pages` у вкладці `Actions`.
5. Повністю закрий Mini App у Telegram і відкрий знову. Старий Service Worker буде автоматично замінено.

## Публічні дані

Публічні файли навмисно порожні, доки немає перевіреної інформації:

- `data/needs.json` — актуальні потреби;
- `data/points.json` — перевірені точки підтримки;
- `data/updates.json` — оголошення.

Приклади структури без автоматичної публікації:

- `examples/needs.example.json`;
- `examples/points.example.json`.

Не копіюй приклад у публічні дані без перевірки адреси, контакту й актуальності.

## Редактор даних

Відкрий:

```text
https://mazapoyt-hash.github.io/KAPTONKA/admin.html
```

Редактор працює локально: він перевіряє JSON і завантажує готовий файл. Після цього файл потрібно вручну замінити в папці `data` на GitHub.

## Telegram

Для статичної Mini App токен бота не потрібен. Адреса Mini App:

```text
https://mazapoyt-hash.github.io/KAPTONKA/
```

Токен не можна додавати в репозиторій, `config.js` або клієнтський JavaScript. Якщо токен уже був опублікований, його потрібно відкликати через BotFather.

## Налаштування

У `config.js` можна вказати:

```js
botUsername: '',
coordinatorTelegram: '',
publicAppUrl: 'https://mazapoyt-hash.github.io/KAPTONKA/',
```

Якщо `coordinatorTelegram` порожній, створене повідомлення можна надіслати через системне меню поширення або вибраний Telegram-чат.

## Перевірка перед публікацією

- у `data` немає вигаданих записів;
- усі адреси й контакти перевірені;
- закриті або застарілі потреби одразу оновлені;
- приватні телефони й адреси не публікуються без згоди;
- токен бота відсутній у репозиторії;
- GitHub Actions завершився без помилок.
